# 珞珈食记 - 单机桌面版 需求文档

## 第一部分：需求文档

### 1. 文档概述

#### 1.1 文档目的

本文档为珞珈食记单机桌面版程序的产品需求说明文件，完整定义产品的业务范围、功能逻辑、交互规则及使用约束，为桌面程序的设计、开发、测试提供统一标准依据。本产品为纯本地运行的桌面工具，无网络依赖、无账号体系、无云端存储，所有功能及数据仅在用户设备本地运行与保存，同时满足课程要求的核心技术点，新增多媒体、正则表达式、多线程/异步编程相关功能。

#### 1.2 产品定位

本产品为面向在校学生的轻量化WinForm桌面工具，聚焦校园饮食管理核心需求，集随机就餐选择、饮食信息归档、消费与热量统计、饭搭子管理、本地趣味数据报告于一体，新增多媒体音效、正则输入校验、多线程/异步编程功能，帮助用户解决就餐选择困难、饮食数据难以追踪的问题，全程无需联网，数据仅存储在用户设备本地，同时贴合课程对技术应用的要求。

#### 1.3 目标用户

核心用户为在校本科生、研究生，有固定食堂就餐习惯，存在饮食消费记账、热量管理、就餐选择纠结等使用需求，偏好无网络依赖、轻量化的本地桌面工具应用；同时适配课程开发要求，作为C# WinForm课程大作业，覆盖面向对象、数据库、Linq、文件操作、多线程/异步编程等核心技术点。

---

### 2. 整体业务规则

- 全量数据仅存储在用户设备本地（基于SQLite数据库），无云端同步、无多设备数据互通，卸载程序、清除本地数据库文件后所有记录将永久丢失；
- 所有就餐记录（随机选餐生成、手动自主录入）统一归档至本地数据库，全部纳入本地数据统计、报告计算范围，无差别对待；
- 食堂、档口、餐食信息、饭搭子等内容均由用户本地自定义新增、编辑、删除，无公共默认数据，所有输入内容通过正则表达式校验，确保格式规范；
- 全程无网络相关功能，无需联网即可使用所有核心功能，无外部接口依赖，多媒体音效仅本地播放；
- 系统内置多维度时间统计周期，涵盖周、月、年度、在校学期四种统计口径，学期起止日期由用户在设置中自定义配置，所有统计数据均基于本地数据库记录，通过Linq查询计算生成；
- 支持本地数据备份/恢复、统计报告导出，所有文件操作均做异常处理，确保功能稳定性；
- 核心操作（老虎机滑动、记录保存、异常提示）配备多媒体音效，提升用户体验；
- 用户上传的照片文件统一复制至程序本地图片目录存储，避免因原始图片移动或删除导致显示异常。

### 3. 详细功能需求

#### 3.1 智能选餐模块（核心功能）

- 采用老虎机上下滑动式随机选择组件（WinForm界面实现），候选档口以文字+图片形式展示，点击"开始滑动"后上下滚动，自动随机减速后定格单一结果；
- **多线程/异步适配**：老虎机滚动动画采用异步编程（async/await）实现，避免动画运行期间UI线程阻塞导致界面卡顿或假死，确保滚动过程中用户仍可操作其他界面元素；
- 多媒体适配：老虎机滑动开始、滑动停止、选择确认时，播放对应本地音效，支持在设置中开启/关闭音效；
- 支持用户提前维护候选列表，可自由添加、移除单个或多个食堂档口，自定义本次选餐的备选范围；
- 支持个性化定制保存选餐范围：用户可创建多个选餐分组（如"常去档口"、"新品尝试"），每个分组包含若干档口，选餐时可快速切换分组，无需每次重复增删候选；
- 支持多次重复随机滚动选择，用户可多次刷新结果，确认最终选择后生成对应的本地就餐记录，记录保存成功时播放提示音效；
- 提供跳过随机选餐入口，用户可手动直接选择食堂、对应档口，上传餐食照片、填写热量数值、消费价格、就餐备注，所有输入内容通过正则表达式校验，手动提交后正常计入本地数据库及统计数据。

#### 3.2 餐食与档口管理

- 支持用户本地自主新增食堂、档口信息，完善档口名称、所属食堂、档口实拍照片等基础信息，所有内容仅保存在本地SQLite数据库；
- 单条餐食可关联对应档口，用户可自定义上传食物照片、填写热量数值、消费金额、口味及用餐备注等细节信息，数据实时写入本地数据库；
- **图片存储策略**：用户上传的照片（档口照片、餐食照片、饭搭子照片）在保存时，自动复制至程序安装目录下的"Images"文件夹，并以"类型_ID_时间戳.扩展名"格式重命名存储，数据库仅保存该本地路径；若用户选择不上传照片，对应字段存空字符串，界面显示默认占位图；
- 正则表达式校验：消费价格（仅允许输入非负数，最多保留2位小数，支持0.5等形式）、热量数值（仅允许输入非负整数或小数）、备注信息（允许中文、字母、数字、常见标点及空格，禁止危险特殊字符，长度限制在200字以内），校验不通过时弹出提示音效及文字提示；
- 已保存的档口、餐食数据可随时编辑、删除、修改，删除操作遵循级联删除策略（详见3.9节），数据变更后实时同步更新本地数据库及历史关联记录，确保数据一致性，操作成功/失败均有对应音效提示。

#### 3.3 饭搭子管理模块

- 独立饭搭子管理页面，用户可本地自主新增多位饭搭子，录入饭搭子姓名、上传个人照片，建立专属本地饭搭子列表，数据存储在本地数据库；
- **"自己"默认选项**：系统在首次启动时自动在数据库中预置一条"自己"饭搭子记录（Id=1, Name="老己"），该记录不可删除、不可重命名，当用户单人独自就餐时，选择该标签标记为单人就餐；
- 正则表达式校验：饭搭子姓名（仅允许输入中文、字母、数字，长度限制在2-20字，禁止特殊字符），校验不通过时弹出提示；
- 每次就餐记录确认环节，增加饭搭子选择项，用户可从已添加的饭搭子列表中**选择一位或多位**同行好友，支持多选；
- 所有共餐关系自动绑定至对应本地就餐记录，通过就餐记录-饭搭子关联表存储，同步写入数据库，纳入后期本地数据统计，支持饭搭子相关的趣味统计功能。

#### 3.4 数据统计分析模块

- 基础统计维度：支持按周、月、年、学期四种周期自动汇总本地就餐相关数据，用户可自由切换查看，所有统计查询通过Linq实现；
- **学期统计配置**：学期起止日期由用户在程序设置中自定义（如2025-09-01至2026-01-15），支持设置当前学期和下学期，未配置学期日期时，学期统计入口灰显并提示用户先配置；
- 展示形式：所有统计数据统一采用表格+饼状图（WinForm Chart控件绘制，满足GUI绘图要求）双呈现模式，表格展示详细明细数据，饼图可视化呈现占比与金额分布；
- 一级统计（食堂维度）：本地展示各食堂数据，饼图呈现各食堂就餐次数占比、消费金额占比，表格展示各食堂的就餐次数、累计消费金额、用餐频次等明细；
- 二级联动统计（档口维度）：点击饼图或表格内的单个食堂，可跳转至该食堂详情统计页，展示该食堂旗下所有档口的就餐次数、占比、消费分布饼图与明细表格；
- 统计内容：涵盖累计消费总额、各食堂就餐频次、各档口就餐频次、周期平均热量、用餐时间分布、饭搭子共餐次数等核心本地数据；
- 多媒体适配：统计图表加载完成、切换统计周期时，播放轻微提示音效，提升交互体验。

#### 3.5 月度报告与学期趣味统计

- 系统基于本地数据库记录，通过Linq查询统计，自动生成月度报告、学期专属报告，用户可在对应周期结束后，主动进入页面查看与导出报告（支持导出为CSV/Excel格式，满足文件操作要求），报告仅本地保存；
- **多线程/异步适配**：报告生成和导出过程采用异步编程实现，避免大数据量导出时UI线程阻塞导致界面假死，导出期间显示进度提示，完成后自动刷新界面；
- 常规数据汇总：报告中包含对应周期内的总用餐次数、总消费金额、热量均值、高频就餐地点等基础数据；
- 趣味数据统计：自动统计周期内就餐次数最高的食堂/档口、消费金额最高的食堂/档口、共餐频次最高的饭搭子，生成本地个性化统计结果；
- 个性化总结：报告整合用户的个性化用餐标签、消费习惯、饮食偏好等内容，以轻量化趣味形式展示，所有内容仅用户本地可见；
- 多媒体适配：报告生成完成、导出成功时，播放对应提示音效，导出失败时播放错误提示音效。

#### 3.6 正则表达式校验功能（新增）

- 核心作用：对用户所有输入内容进行格式校验，确保数据规范性，减少异常数据录入，降低数据库读写错误风险；
- 校验范围及规则：
  - 消费价格：匹配非负数，允许整数或最多2位小数（如0、0.5、10、12.5、8.36），禁止负数、字母、特殊字符；
  - 热量数值：匹配非负整数或小数（如0、500、320.5），禁止负数、字母、特殊字符；
  - 饭搭子姓名：匹配中文、字母、数字，长度2-20字，禁止特殊字符（如@、#、&等）；
  - 备注信息：允许中文、字母、数字、常见标点（，。！？、；：""''（）【】《》·~—…）、空格，禁止危险特殊字符（如<、>、&、'、"等可能引发注入风险的字符），长度限制在200字以内；
  - 食堂/档口名称：匹配中文、字母、数字、空格、括号（（）），长度2-50字，禁止其他特殊字符；
- 校验反馈：校验不通过时，弹出文字提示（如"请输入有效的消费价格，最多保留2位小数"），同时播放错误提示音效，聚焦至错误输入框，引导用户修正。

#### 3.7 多媒体功能（新增）

- 音效类型及触发场景：
  - 操作成功音效：记录保存、数据修改/删除成功、报告导出成功、数据备份/恢复成功时播放；
  - 操作失败/异常音效：输入校验不通过、数据库读写失败、文件操作失败、无候选档口时点击选餐时播放；
  - 交互音效：老虎机滑动开始、滑动停止、选择确认、切换统计周期、图表加载完成时播放；
  - 提示音效：程序启动、退出时播放简短音效；
- 音效设置：支持在程序设置页面开启/关闭所有音效，或单独关闭某类音效（如交互音效），设置实时生效并保存至本地配置文件；
- 音效存储：所有音效文件均打包在程序安装目录下，本地读取播放，无需联网下载，支持常见音频格式（如wav）。

#### 3.8 多线程/异步编程功能（新增）

- 核心作用：通过async/await异步编程模式，避免耗时操作阻塞UI线程，保证程序运行流畅、界面响应及时，满足课程可选项要求；
- 应用场景：
  - 老虎机滚动动画：异步执行滚动动画逻辑，滚动期间UI线程保持响应，用户可随时中断或操作其他界面元素；
  - 报告生成与导出：异步执行Linq统计计算和文件写入操作，大数据量导出时界面不卡顿，显示进度提示；
  - 数据备份/恢复：异步执行文件复制操作，大文件复制期间界面保持响应，显示复制进度；
  - 图表数据加载：异步执行Linq查询并绑定Chart控件数据，避免统计页面切换时界面短暂卡死；
- 实现方式：在BLL层对应方法上添加async修饰符，返回Task或Task<T>，UI层调用时使用await等待结果，期间UI线程可处理其他事件。

#### 3.9 数据删除与级联策略

为保障数据一致性，定义以下级联删除规则：

- 删除食堂 → 级联删除该食堂下所有档口 → 级联删除各档口下所有餐食 → 级联删除关联的就餐记录及就餐记录-饭搭子关联数据，操作前弹出确认提示，明确告知影响范围；
- 删除档口 → 级联删除该档口下所有餐食 → 级联删除关联的就餐记录及就餐记录-饭搭子关联数据，操作前弹出确认提示；
- 删除餐食 → 就餐记录中MealId置空（保留记录，因就餐记录可能仅记录档口级别信息），操作前弹出确认提示；
- 删除饭搭子 → 就餐记录-饭搭子关联表中移除该饭搭子的所有关联行（就餐记录保留，仅解除共餐关系），若为"自己"记录则禁止删除；
- 所有级联操作在同一个数据库事务内执行，任一步骤失败则整体回滚，确保数据一致性。

#### 3.10 辅助功能（满足课程可选项）

- 数据备份/恢复：支持一键备份本地SQLite数据库文件至用户指定路径，也可从备份文件恢复数据，操作过程中做异常处理，备份/恢复成功/失败均有音效提示；
- 报告导出：将月度/学期报告、统计数据导出为CSV/Excel格式，保存至本地，导出过程中做异常处理（如权限不足、路径不存在），并给出对应提示；
- 异常处理：所有关键流程（输入校验、数据库读写、文件操作、图表渲染、音效播放）均做异常捕获，弹出友好提示，避免程序闪退；
- 程序设置：支持设置音效开关、默认统计周期、报告导出默认路径、学期起止日期，所有设置保存至本地配置文件，程序重启后生效。

### 4. 非功能性需求

#### 4.1 本地数据性能

- 核心页面加载、老虎机滑动选餐动画、统计图表渲染响应流畅，无明显卡顿，本地数据库读写、Linq查询响应时间不超过1秒；
- 本地数据基于SQLite数据库持久化存储，支持用户手动备份/导出数据库文件，避免设备异常导致数据丢失；
- 程序重启后，所有本地数据、用户设置可正常读取，无数据错乱、丢失问题；
- 音效播放流畅，无卡顿、延迟，与操作同步，不影响程序核心功能运行；
- 异步操作期间UI保持响应，不出现界面假死或无响应状态。

#### 4.2 兼容性

适配主流Windows 10/11系统，界面自适应常见桌面分辨率（如1920×1080、1366×768），确保不同设备上的基础操作体验统一；支持32位、64位Windows系统，无需额外安装依赖（SQLite打包至程序内）。

#### 4.3 易用性

整体界面采用简洁轻量化设计，功能分区清晰，核心的选餐、统计、饭搭子管理等功能入口直观易找，操作逻辑简单易懂，无需复杂学习即可上手；正则校验提示、异常提示清晰明确，音效反馈贴合操作场景，提升用户体验。

#### 4.4 稳定性

所有关键流程均添加异常处理，避免程序闪退；数据库操作做事务处理，防止数据写入失败导致的数据错乱；文件操作做权限校验、路径校验，确保功能稳定运行；音效播放失败时不影响其他功能，给出友好提示；异步操作异常时自动回退至同步模式或给出提示。

---

### 5. 业务约束

- 程序为纯单机WinForm桌面应用，无任何网络功能、无账号体系、无云端存储，所有数据仅存储在用户设备本地SQLite数据库中；
- 卸载程序、清除本地数据库文件或配置文件，将导致所有用户数据、设置永久丢失，无法恢复；
- 所有统计数据均以用户本地提交的就餐记录为唯一计算依据，手动录入与随机选餐生成的记录权重一致，无优先级差异；
- 食堂、档口、饭搭子等自定义内容，均由用户自主维护，系统不提供任何公共数据或外部内容（"自己"默认饭搭子除外）；
- 正则表达式校验为强制校验，输入内容不满足规则时，无法完成对应操作（如保存记录、添加饭搭子）；
- 音效文件需打包至程序安装目录，删除音效文件将导致音效无法播放，不影响核心功能运行；
- 用户上传的照片统一复制至本地图片目录，原始图片的移动或删除不影响程序内图片显示。
