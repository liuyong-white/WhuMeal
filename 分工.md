人员分工详情
------

### 成员 A-刘咏：UI 交互与核心功能（智能选餐 + 多媒体）

#### 核心负责模块

1. **智能选餐模块（全量开发）**
   
   * 老虎机滑动组件 WinForm 界面实现（文字 + 图片展示、滚动动画逻辑）；
   * 异步编程适配：基于 async/await 实现老虎机滚动动画，避免 UI 阻塞；
   * 选餐分组管理（新增 / 切换 / 删除分组、维护候选档口列表）；
   * 手动选餐入口开发（食堂 / 档口选择、数据录入界面）；
   * 选餐结果确认及就餐记录生成逻辑。

2. **多媒体功能（全量开发）**
   
   * 所有场景音效（操作成功 / 失败、交互、提示类）的本地播放逻辑；
   * 音效设置页面开发（全局开关、分类开关）；
   * 音效与核心操作的联动（如选餐确认、校验失败时触发音效）；
   * 音效文件的本地读取、异常处理（如文件缺失时的兼容）。

3. **通用支撑：界面统一风格与基础控件封装**
   
   * 制定 WinForm 界面统一样式（按钮、输入框、弹窗、占位图等）；
   * 封装图片展示控件（适配本地图片路径、默认占位图逻辑）；
   * 核心页面的布局适配（兼容 1920×1080、1366×768 分辨率）。

#### 交付物

* 智能选餐模块完整界面 + 逻辑代码；
* 多媒体音效播放 / 设置功能代码；
* 全局 UI 样式规范及基础控件封装库；
* 模块内异常处理（如无候选档口、音效播放失败）。

### 成员 B-蔡爽：数据层与基础管理（档口 / 餐食 / 饭搭子 + 正则 + SQLite）

#### 核心负责模块

1. **餐食与档口管理 + 饭搭子管理模块（全量开发）**
   
   * 食堂 / 档口 / 餐食的增删改查界面及逻辑；
   * 饭搭子管理（新增 / 编辑 / 删除、“自己” 默认记录的初始化）；
   * 照片上传与本地存储逻辑（复制至 Images 文件夹、重命名、数据库存路径）；
   * 就餐记录与饭搭子的关联存储（多选饭搭子、关联表维护）。

2. **正则表达式校验功能（全量开发）**
   
   * 所有输入项的正则规则实现（价格、热量、姓名、备注、食堂 / 档口名称）；
   * 校验失败的提示逻辑（文字提示 + 音效 + 输入框聚焦）；
   * 正则校验与表单提交的联动（校验不通过禁止保存）。

3. **数据层核心（SQLite 数据库设计与封装）**
   
   * 数据库表结构设计（食堂、档口、餐食、就餐记录、饭搭子、关联表等）；
   * SQLite 连接封装、增删改查通用方法（DAL 层）；
   * 级联删除规则实现（数据库事务保障一致性）；
   * 数据备份 / 恢复的底层文件操作逻辑。

#### 交付物

* 档口 / 餐食 / 饭搭子管理模块完整代码；
* 正则校验工具类 + 所有输入校验逻辑；
* SQLite 数据库设计文档 + DAL 层封装代码；
* 图片 / 数据库文件的本地操作逻辑。

### 成员 C-齐赫然：统计分析与异步拓展（统计 + 报告 + 异步）

#### 核心负责模块

1. **数据统计分析模块（全量开发）**
   
   * 多周期统计（周 / 月 / 年 / 学期）的 Linq 查询逻辑；
   * 统计页面开发（表格 + Chart 饼图展示、食堂 / 档口二级联动）；
   * 学期日期配置页面开发；
   * 统计数据的可视化渲染（Chart 控件使用）。

2. **月度 / 学期报告模块（全量开发）**
   
   * 报告生成逻辑（Linq 统计计算、常规 / 趣味数据汇总）；
   * 异步编程适配：报告生成 / 导出的 async/await 实现 + 进度提示；
   * 报告导出功能（CSV/Excel 格式、本地保存、异常处理）；
   * 个性化总结与趣味统计逻辑。

3. **通用支撑：辅助功能 + 全局异常处理**
   
   * 程序设置页面（音效开关、默认统计周期、导出路径、学期配置）；
   * 全局异常处理（数据库读写、文件操作、异步操作异常捕获）；
   * 所有模块的文件操作封装（备份 / 恢复、导出）。

#### 交付物

* 统计分析模块完整代码；
* 报告生成 / 导出模块完整代码；
* 程序设置页面 + 全局异常处理代码；
* Linq 查询工具类、文件操作封装类。

协作与联调规则
-------

1. **接口规范先行**：成员 B 完成数据库表结构和 DAL 层封装后，输出接口文档，成员 A、C 基于统一接口开发业务逻辑；

2. **每日同步进度**：聚焦模块内卡点（如异步动画卡顿、正则规则兼容、Chart 图表渲染问题），及时协调；

3. **联调阶段（最后 1/3 周期）**
   
   * 成员 A：集成多媒体音效到成员 B/C 的模块（如校验失败、报告导出成功时触发音效）；
   * 成员 B：保障所有模块的数据库操作一致性（级联删除、事务回滚）；
   * 成员 C：验证统计 / 报告模块能正确读取选餐、饭搭子模块的数据源；

4. **测试分工**：每人除测试自己模块外，交叉测试另外两人的核心模块（如 A 测统计模块、B 测选餐模块、C 测饭搭子模块）。

里程碑节点
-----

1. 第一阶段（1/3 周期）：成员 B 完成数据库 + 正则校验；成员 A 完成选餐界面框架；成员 C 完成统计 Linq 查询框架；
2. 第二阶段（2/3 周期）：每人完成核心模块开发，内部自测；
3. 第三阶段（1/3 周期）：联调 + 全量测试 + Bug 修复 + 文档完善
