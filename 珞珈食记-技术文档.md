# 珞珈食记 - 单机桌面版 技术文档

## 1. 文档概述

### 1.1 设计目的

基于需求文档，明确珞珈食记单机桌面版（WinForm）的技术架构、数据库设计、界面设计、功能实现逻辑、核心技术点应用，指导开发人员进行编码开发，确保产品功能符合需求，同时满足课程要求的面向对象、数据库、Linq、文件操作、GUI绘图、多媒体、正则表达式、多线程/异步编程等技术点，确保程序稳定性、易用性达标。

### 1.2 设计原则

- 实用性：优先实现核心功能，简化技术实现，降低开发难度，贴合课程要求，优先应用指定技术点；
- 可维护性：代码结构清晰，采用面向对象思想设计，类与类之间低耦合、高内聚，便于后期修改和扩展；
- 稳定性：所有关键流程添加异常处理，数据库操作、文件操作、音效播放均做容错处理，避免程序闪退；
- 易用性：界面设计简洁直观，交互流程流畅，贴合桌面程序操作习惯，正则校验、异常提示清晰；
- 轻量化：采用轻量级技术选型（SQLite、WinForm自带控件），降低程序体积，确保运行流畅，无需额外安装依赖。

## 2. 总体架构设计

### 2.1 架构选型

采用"界面层（UI）+ 业务逻辑层（BLL）+ 数据访问层（DAL）+ 数据存储层"四层架构，基于C# WinForm开发，全程本地运行，无网络依赖，各层职责清晰，贴合面向对象设计思想，具体架构如下：

- **界面层（UI）**：基于WinForm实现，负责程序界面展示、用户交互，包含所有窗体、控件（按钮、输入框、表格、Chart图表等），同时负责多媒体音效播放调用、正则表达式输入校验触发、异步操作进度展示；
- **业务逻辑层（BLL）**：负责核心业务逻辑处理，如老虎机随机选择算法、统计数据计算（Linq查询）、报告生成、异常处理逻辑，协调UI层与DAL层的数据交互，核心耗时方法采用async/await异步实现；
- **数据访问层（DAL）**：负责与本地SQLite数据库交互，实现数据的增删改查操作，封装数据库访问方法，提供给BLL层调用，确保数据操作的统一性和安全性；
- **数据存储层**：采用SQLite轻量级本地数据库，存储所有用户数据（食堂、档口、就餐记录、饭搭子等），同时包含本地配置文件（存储音效设置、导出路径、学期日期等）、音效文件、图片文件。

### 2.2 核心技术栈

- 开发语言：C#（.NET Framework 4.8，贴合课程教学内容）；
- 界面框架：WinForm（Windows Forms），使用自带控件（Button、TextBox、DataGridView、Chart等）实现界面；
- 数据库：SQLite（轻量级本地数据库，无需安装服务，打包至程序内，支持本地持久化存储）；
- 数据库访问：System.Data.SQLite（SQLite的C#驱动）+ Dapper（简化数据库操作，提升开发效率）；
- 数据查询：Linq（用于统计数据查询、数据筛选、分组统计，满足课程可选项要求）；
- GUI绘图：WinForm自带Chart控件（绘制饼图、表格，满足GUI绘图要求）；
- 正则表达式：C# Regex类，实现用户输入内容校验；
- 多媒体：System.Media.SoundPlayer（播放本地音效文件）；
- 文件操作：System.IO命名空间，实现报告导出（CSV/Excel）、数据备份/恢复、配置文件读写、图片文件复制；
- 异常处理：try-catch异常捕获机制，针对不同场景（输入、数据库、文件、音效）做针对性异常处理；
- 多线程/异步编程：async/await模式（基于Task），用于老虎机动画、报告导出、数据备份恢复、图表加载等耗时操作，满足课程可选项要求。

### 2.3 架构流程图

用户操作 → UI层（接收操作、触发正则校验、调用音效）→ BLL层（处理业务逻辑、Linq查询统计、异步执行耗时操作）→ DAL层（数据库增删改查）→ 数据存储层（SQLite数据库/本地文件）→ DAL层返回结果 → BLL层处理结果 → UI层展示结果/播放对应音效。

## 3. 面向对象设计（类设计）

采用面向对象思想，封装核心实体类、业务逻辑类、数据访问类，实现类的封装、复用，贴合课程要求，具体类设计如下：

### 3.1 实体类（Model层）

封装核心数据实体，对应数据库表结构，属性与表字段一一对应，提供get/set方法，实现数据封装。

- **Canteen（食堂类）**：属性包括Id（主键）、CanteenName（食堂名称）、IsSystem（是否系统预置），封装食堂相关信息；IsSystem=TRUE表示武汉大学预置食堂（未被用户修改），仅用于来源标记和界面区分展示，可编辑/删除，编辑后IsSystem自动改为FALSE；
- **Stall（档口类）**：属性包括Id（主键）、StallName（档口名称）、CanteenId（关联食堂Id）、StallPhoto（档口照片路径）、IsSystem（是否系统预置），关联食堂类；IsSystem=TRUE表示武汉大学预置档口（未被用户修改），仅用于来源标记，可编辑/删除，编辑后IsSystem自动改为FALSE；
- **Meal（餐食类）**：属性包括Id（主键）、MealName（餐食名称）、StallId（关联档口Id）、MealPhoto（餐食照片路径）、Calorie（热量）、Price（价格）、Remark（备注）、IsSystem（是否系统预置），关联档口类；MealName为空时，由BLL层在查询时自动补全为关联档口名称，UI层直接使用BLL返回的结果；IsSystem=TRUE表示系统预置餐食（未被用户修改），可编辑/删除，编辑后IsSystem自动改为FALSE；
- **DinnerBuddy（饭搭子类）**：属性包括Id（主键）、Name（姓名）、Photo（照片路径）、IsSystem（是否系统预置，如"自己"），封装饭搭子信息；
- **MealRecord（就餐记录类）**：属性包括Id（主键）、StallId（关联档口Id）、MealId（关联餐食Id，可空）、RecordTime（记录时间）、Calorie（热量）、Price（价格）、Remark（备注），StallId通过Meal.StallId关联推导，同时冗余存储以支持餐食删除后仍保留档口信息；
- **MealRecordBuddy（就餐记录-饭搭子关联类）**：属性包括RecordId（关联就餐记录Id）、BuddyId（关联饭搭子Id），实现一条就餐记录对应多位饭搭子的多对多关系；
- **SelectionGroup（选餐分组类）**：属性包括Id（主键）、GroupName（分组名称），封装用户自定义的选餐范围分组；
- **SelectionGroupStall（选餐分组-档口关联类）**：属性包括GroupId（关联分组Id）、StallId（关联档口Id），实现分组与档口的多对多关系；
- **AppSetting（应用设置类）**：属性包括SoundEnabled（音效总开关）、InteractiveSoundEnabled（交互音效开关）、ExportPath（默认导出路径）、SemesterStartDate（学期开始日期）、SemesterEndDate（学期结束日期），封装程序设置信息。

### 3.2 业务逻辑类（BLL层）

- **MealSelectBLL（选餐业务类）**：负责老虎机随机选择算法、选餐逻辑处理、选餐分组管理，核心方法采用async修饰（如`async Task<Stall> RandomSelectAsync()`）；
- **DataManageBLL（数据管理类）**：负责食堂、档口、餐食、饭搭子的新增、编辑、删除逻辑，级联删除逻辑，预置数据编辑时自动将IsSystem置FALSE，正则表达式校验逻辑；
- **StatisticBLL（统计业务类）**：负责统计数据计算（Linq查询）、图表数据组装、月度/学期报告生成，统计方法采用async修饰；
- **FileOperateBLL（文件操作类）**：负责报告导出（CSV/Excel）、数据备份/恢复、配置文件读写、图片文件复制管理，文件操作方法采用async修饰；
- **SoundBLL（音效业务类）**：负责音效文件加载、播放、关闭，音效开关控制，播放方法采用异步调用避免阻塞UI线程。

### 3.3 数据访问类（DAL层）

- **BaseDAL（基础数据访问类）**：封装数据库连接、基础增删改查方法，供其他DAL类继承，减少代码冗余；封装事务执行方法`ExecuteInTransaction(Action action)`，确保级联操作的原子性；
- **CanteenDAL（食堂数据访问类）**：负责食堂相关数据的增删改查操作，包含系统预置食堂数据的初始化写入方法；
- **StallDAL（档口数据访问类）**：负责档口相关数据的增删改查操作，包含系统预置档口数据的初始化写入方法；
- **MealDAL（餐食数据访问类）**：负责餐食相关数据的增删改查操作，包含系统预置餐食数据的初始化写入方法；
- **DinnerBuddyDAL（饭搭子数据访问类）**：负责饭搭子相关数据的增删改查操作，包含"自己"默认记录的初始化检查方法；
- **MealRecordDAL（就餐记录数据访问类）**：负责就餐记录相关数据的增删改查操作；
- **MealRecordBuddyDAL（就餐记录-饭搭子关联数据访问类）**：负责就餐记录与饭搭子多对多关联的增删改查操作；
- **SelectionGroupDAL（选餐分组数据访问类）**：负责选餐分组及分组-档口关联的增删改查操作；
- **ConfigRepository（配置存储类）**：负责配置文件的读写操作（JSON格式），存储音效开关、导出路径、学期日期等设置信息，独立于数据库操作，命名明确区分于数据库访问层。

### 3.4 辅助类（Helper层）

- **RegexHelper（正则校验辅助类）**：封装各类输入校验的正则表达式及校验方法，返回校验结果（bool）和提示信息（string），供BLL层和UI层调用；
- **ExceptionHelper（异常处理辅助类）**：封装全局异常捕获处理逻辑、异常提示信息格式化，在Program.cs中注册Application.ThreadException事件时调用。

## 5. 核心功能技术实现

### 5.1 系统预置数据初始化

- **首次启动检测**：程序启动时，在BaseDAL类中封装`void EnsureSystemDataInitialized()`方法，检测Canteen表中是否存在IsSystem=TRUE的记录，若不存在则执行预置数据写入；
- **预置数据写入**：在CanteenDAL、StallDAL中分别封装`void InsertSystemCanteens()`和`void InsertSystemStalls()`方法，将武汉大学全部食堂及档口基础信息（含IsSystem=TRUE标记）批量写入数据库，写入操作在同一个事务内执行，确保数据完整性；
- **预置数据清单**：武汉大学食堂及档口的完整清单在开发阶段确定，硬编码在DAL层的初始化方法中（以常量数组或嵌入资源形式），随程序一起发布，具体食堂与档口列表在实现代码阶段提供；
- **IsSystem语义**：IsSystem=TRUE仅表示数据来源为系统预置且未被用户修改，用于界面区分展示（标记"内置"标签）；用户编辑预置数据时，BLL层自动将IsSystem置为FALSE，表示该条目已脱离原始预置状态；所有数据（含预置）均可自由编辑、删除。

### 5.2 智能选餐模块实现（老虎机式）

- **界面实现**：使用WinForm的Panel、PictureBox、Label控件搭建老虎机界面，PictureBox展示档口图片，Label展示档口名称，通过Timer控件实现上下滑动动画；
- **异步动画实现**：在MealSelectBLL类中，将滚动动画核心逻辑封装为`async Task AnimateRollAsync()`，使用await Task.Delay()替代Thread.Sleep()控制减速节奏，UI层通过`await mealSelectBLL.AnimateRollAsync()`调用，滚动期间UI线程保持响应；
- **随机选择算法**：在BLL层MealSelectBLL类中，通过Random类生成随机索引，从候选档口列表中随机选择一个档口，滑动动画结束后定格该档口；
- **选餐分组实现**：用户在选餐界面选择已保存分组后，从SelectionGroupStall关联表查询该分组包含的档口Id列表，加载对应档口作为候选范围；
- **多媒体实现**：在SoundBLL类中，封装SoundPlayer对象，加载对应音效文件（如滑动开始、停止、确认），在老虎机操作对应节点调用Play()方法播放音效，支持暂停、停止播放；
- **跳过选餐实现**：添加"直接录入"按钮，点击后跳转至录入窗体，手动选择食堂、档口，录入相关信息，校验通过后保存至数据库。

### 5.2 正则表达式校验实现

- **封装校验方法**：在RegexHelper类中，封装正则校验方法，针对不同输入类型（价格、热量、姓名、备注、名称）编写对应正则表达式，返回校验结果（bool）和提示信息（string）；
- **核心正则表达式**：
  - 消费价格：`^(0|[1-9]\d*)(\.\d{1,2})?$`（匹配非负数，整数部分允许0，最多2位小数，如0、0.5、10、12.50）；
  - 热量数值：`^[0-9]+(\.\d+)?$`（匹配非负整数或小数）；
  - 饭搭子姓名：`^[a-zA-Z0-9\u4e00-\u9fa5]{2,20}$`（匹配2-20字中文、字母、数字）；
  - 备注：`^[a-zA-Z0-9\u4e00-\u9fa5\s，。！？、；：""''（）【】《》·~—…]{1,200}$`（允许中文/字母/数字/常见标点/空格，1-200字）；危险字符黑名单校验：`[<>&'"]`（匹配需额外拦截的注入风险字符）；
  - 食堂/档口名称：`^[a-zA-Z0-9\u4e00-\u9fa5\s（）]{2,50}$`（匹配中文/字母/数字/空格/括号，2-50字）；
- **校验触发**：在UI层输入框的LostFocus事件（失去焦点）或按钮的Click事件（提交）中，调用RegexHelper校验方法，校验不通过则弹出提示、播放错误音效，聚焦输入框；
- **备注双重校验**：备注字段先做长度+白名单格式校验，再额外做危险字符黑名单校验（防止SQL注入/HTML注入），两项均通过才算校验成功。

### 5.3 多媒体音效实现

- **音效文件管理**：将所有音效文件（wav格式）打包至程序安装目录下的"Sound"文件夹，在SoundBLL类中，通过Application.StartupPath获取程序路径，拼接音效文件路径；
- **音效播放封装**：在SoundBLL类中，封装`async Task PlaySoundAsync(string soundPath)`方法，创建SoundPlayer对象，加载音效文件，异步调用Play()方法播放，避免阻塞UI线程；封装StopSound()方法，停止当前音效播放；
- **音效开关控制**：在AppSetting类中存储音效开关状态，在SoundBLL类中，播放音效前判断开关状态，开启则播放，关闭则不播放；用户可在设置窗体修改开关状态，实时生效并保存至ConfigRepository配置文件；
- **异常处理**：音效播放时，捕获FileNotFoundException（文件不存在）、InvalidDataException（文件格式错误），弹出提示，不影响其他功能运行。

### 5.4 数据统计模块实现（Linq应用）

- **数据查询**：在StatisticBLL类中，通过DAL层获取所有就餐记录，使用Linq查询实现数据筛选、分组、统计：
  - 按时间筛选：`var weekRecords = mealRecords.Where(r => r.RecordTime >= startDate && r.RecordTime <= endDate).ToList();`
  - 学期筛选：从ConfigRepository读取学期起止日期，同上逻辑筛选；
  - 食堂分组统计：`var canteenStats = mealRecords.GroupBy(r => r.Stall.CanteenId).Select(g => new { CanteenName = g.First().Stall.Canteen.CanteenName, Count = g.Count(), TotalPrice = g.Sum(r => r.Price), AvgCalorie = g.Average(r => r.Calorie) }).ToList();`
  - 饭搭子共餐统计：`var buddyStats = mealRecordBuddies.GroupBy(b => b.BuddyId).Select(g => new { BuddyName = g.First().DinnerBuddy.Name, Count = g.Select(b => b.RecordId).Distinct().Count() }).OrderByDescending(g => g.Count).ToList();`（基于MealRecordBuddy关联表统计）；
- **图表实现**：使用WinForm自带的Chart控件，在UI层统计窗体中，将Linq查询得到的统计数据绑定至Chart控件，设置图表类型为饼图（占比）、柱状图（明细），自定义图表颜色、标题、图例；
- **异步图表加载**：统计方法封装为`async Task<StatisticResult> CalculateAsync(Period period)`，UI层调用后await等待，期间显示加载提示，避免界面卡死；
- **二级联动**：点击Chart控件中的食堂饼图扇区，触发Click事件，获取对应食堂Id，通过Linq查询该食堂下所有档口的统计数据，重新绑定至Chart控件，展示档口维度统计。

### 5.5 月度/学期报告实现

- **报告数据计算**：在StatisticBLL类中，根据用户选择的周期（月度/学期），通过Linq查询计算常规数据（总次数、总消费等）和趣味数据（最爱食堂、最佳饭搭子等），方法封装为async；
- **报告展示**：在UI层报告窗体中，用Label、DataGridView控件展示报告内容，排版简洁清晰，突出趣味统计结果；
- **报告导出**：在FileOperateBLL类中，封装`async Task ExportReportAsync(string filePath, ReportData data)`方法，将报告数据写入CSV/Excel文件（使用System.IO命名空间操作文件），导出路径可自定义，导出成功后播放提示音效，失败则捕获异常并提示；导出期间UI显示进度提示（如"正在导出..."），通过IProgress<T>报告进度。

### 5.6 数据备份/恢复实现（文件操作）

- **数据备份**：在FileOperateBLL类中，封装`async Task BackupAsync(string targetPath)`方法，备份前先关闭所有SQLite连接（调用SQLiteConnection.ClearAllPools()释放连接池），再使用File.Copy()方法将数据库文件复制到用户指定的备份路径，备份成功后播放提示音效；
- **数据恢复**：封装`async Task RestoreAsync(string sourcePath)`方法，同样先关闭所有SQLite连接，使用File.Copy()方法将用户选择的备份文件覆盖当前数据库文件，恢复前弹出确认提示，恢复成功后重新初始化数据库连接并刷新所有数据，播放提示音效；
- **异常处理**：捕获FileNotFoundException（文件不存在）、IOException（权限不足、文件被占用），弹出对应提示，确保操作安全。

### 5.7 图片文件管理实现

- **图片复制存储**：在FileOperateBLL类中，封装`string CopyImageToLocalStorage(string sourcePath, string category, int entityId)`方法，将用户选择的图片文件复制至程序安装目录下"Images/{category}"文件夹，以"{category}_{entityId}_{timestamp}{extension}"格式重命名，返回新路径存入数据库；
- **图片删除**：删除档口/餐食/饭搭子时，同步删除对应本地图片文件（若存在），在级联删除的事务外执行，图片删除失败不影响数据删除结果；
- **默认占位图**：程序安装目录下预置默认占位图（如"default_stall.png"），当数据库照片路径为空或文件不存在时，UI层显示默认占位图。

### 5.8 级联删除实现

- **事务封装**：在BaseDAL类中，封装`void ExecuteInTransaction(Action action)`方法，使用SQLiteTransaction包裹操作，任一步骤失败则整体回滚；
- **删除食堂**：在DataManageBLL类中，`async Task DeleteCanteenAsync(int canteenId)`方法，在事务内依次：删除SelectionGroupStall中关联档口的关联行 → 删除MealRecordBuddy中关联记录的关联行 → 删除MealRecord中关联记录 → 删除Meal中关联餐食 → 删除Stall中关联档口 → 删除Canteen记录；
- **删除档口**：`async Task DeleteStallAsync(int stallId)`方法，同上逻辑，范围缩至该档口及下属；
- **删除餐食**：`async Task DeleteMealAsync(int mealId)`方法，将关联MealRecord的MealId置NULL（保留记录），删除Meal记录；
- **删除饭搭子**：`async Task DeleteDinnerBuddyAsync(int buddyId)`方法，检查IsSystem字段，若为TRUE则抛出异常拒绝删除，否则删除MealRecordBuddy中该BuddyId的所有关联行，删除DinnerBuddy记录；
- **操作前确认**：UI层在触发级联删除前，弹出确认对话框，显示影响范围（如"将同时删除该食堂下3个档口、12条就餐记录，确认删除？"），用户确认后调用BLL层方法。

### 5.9 异常处理实现

- **全局异常捕获**：在程序入口（Program.cs）中，注册Application.ThreadException事件和AppDomain.CurrentDomain.UnhandledException事件，捕获UI线程和非UI线程异常，统一调用ExceptionHelper类的处理方法，弹出友好提示，避免程序闪退；
- **局部异常捕获**：在各层关键方法（数据库操作、文件操作、音效播放、正则校验）中，添加try-catch捕获特定异常，如SQLiteException（数据库错误）、IOException（文件错误）、FileNotFoundException（文件不存在），针对性给出提示；
- **异步异常处理**：async方法中的异常通过await捕获，或在Task.ContinueWith中处理，确保异步操作异常不会逃逸至全局；
- **异常日志**：可选补充，将异常信息（时间、异常类型、异常信息、堆栈跟踪）写入本地日志文件（Logs/error_{date}.txt），便于后期调试。

## 6. 界面设计（UI）

### 6.1 界面整体风格

简洁、轻量化、校园风，主色调采用浅蓝、白色，辅助色采用橙色，界面布局清晰，控件大小适中，贴合桌面程序操作习惯；字体清晰易读，按钮、输入框交互反馈明确。

### 6.2 主要界面模块

- **主界面**：侧边栏导航，包含"智能选餐"、"数据管理"、"统计分析"、"趣味报告"、"饭搭子管理"、"设置"六个入口，主区域展示今日就餐概览（已记录次数、今日消费、今日热量）；
- **智能选餐界面**：中央老虎机滚动区域，左侧候选档口列表（可增删），右侧选餐分组切换，底部"开始选餐"和"直接录入"按钮；
- **数据管理界面**：Tab页签切换食堂/档口/餐食管理，各Tab内上部为新增表单，下部为数据列表（DataGridView），支持行内编辑和删除；未修改的预置数据行（IsSystem=TRUE）标记"内置"标签以区分来源，所有行均可正常编辑/删除；
- **统计分析界面**：顶部周期切换栏（周/月/年/学期），左侧饼图区域，右侧明细表格区域，点击饼图扇区触发二级联动；
- **趣味报告界面**：报告列表页（月度/学期），点击进入报告详情页，详情页展示常规统计+趣味统计+个性化总结，底部"导出报告"按钮；
- **饭搭子管理界面**：卡片式展示饭搭子列表，每张卡片显示照片+姓名，支持新增/编辑/删除，"自己"卡片置顶且标有系统标签；
- **设置界面**：音效开关（总开关+分类开关）、默认导出路径、学期起止日期配置、数据备份/恢复操作区。

### 6.3 响应式适配

- 界面使用Anchor和Dock属性实现控件自适应，确保在1366×768和1920×1080分辨率下布局合理；
- Chart控件尺寸根据窗体大小自动调整，数据标签字体大小自适应。
